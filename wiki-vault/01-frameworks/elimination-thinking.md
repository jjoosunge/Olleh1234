# 인원 유리 상황에서 라인 합류 판단

> ⚡ 더블킬 등 인원 유리 발생 시 어느 라인으로 합류할지 판단하는 프레임워크

**Category**: framework  
**Tags**: position:any, phase:mid-game, topic:macro  
**Related**: [[loss-minimization]]

---

## 🎯 적용 시점

팀이 인원 유리를 만들었을 때 (예: 한타 승리, 더블킬, 적 1~2명 처치).
이 시점에 "다음 어디로 갈지" 결정해야 하는 모든 순간.

## 🧠 핵심 사고 흐름 — 선택지 제거 사고법

목표 라인을 "왜 가야 하는지"가 아니라 **"왜 다른 곳을 안 가는지"** 로 답을 도출.

순서대로 제거:

1. **바텀 체크**
   - 원딜이 혼자 충분히 라인 압박 가능한가?
   - 와드/시야 충분한가? 원딜이 안전한가?
   - YES면 → 바텀 제거 (지원 불필요)
   - NO면 → 바텀이 답일 가능성

2. **탑 체크**
   - 거리가 가까운가?
   - 거기 가서 추가 이득을 만들 수 있는가?
   - NO면 → 탑 제거

3. **남은 선택지**
   - 바텀과 탑이 모두 제거됐으면 → 미드가 답

## 🔀 조건부 분기

판단 후에도 상황은 변함.

**팀원 합류 가능한 상태:**
- 적 픽업 시도 가능 (예: 적 서포터)
- 오브젝트 강제 가능

**팀원 합류 불가능 (이미 사망 등):**
- 절대 원칙: "팀이 다시 모이기 전까지는 더 잃지 않는다"
- 수비 시야만 깔고 손해 회피 모드
- 추가 무리 금지

## ❌ 흔한 실수

1. 더블킬 났다고 무리하게 추가 이득 노리기
2. 단일 정답에 집착 (한 가지 선택만 옳다고 보는 사고)
3. 결과론으로 판단 (지나간 결과 보고 평가)
4. 합류 없이 단독 행동

## ✅ 교정 방법

머릿속 체크리스트를 강제:
1. 인원 유리 발생 → 즉시 맵 확인 (정지 X, 이동하면서)
2. 각 라인 제거법 순차 적용
3. 결정된 라인으로 이동 시 팀 합류 상태 추적
4. 합류 불가 시 → 손실 최소화 모드 전환
